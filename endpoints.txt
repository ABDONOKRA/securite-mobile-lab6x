RISQUES IDENTIFIÉS:
-------------------
1. Trafic mixte HTTP/HTTPS
   → Risque de downgrade attack
   → Certificate pinning recommandé

2. URL HTTP hardcodée
   → Interception du trafic possible
   → Données sensibles potentiellement exposées

3. Endpoint externe (GitHub raw content)
   → Dépendance à une ressource externe
   → Possible manipulation du contenu

RECOMMANDATIONS:
----------------
1. Éliminer toute URL HTTP
2. Implémenter certificate pinning pour endpoints critiques
3. Utiliser network_security_config pour bloquer cleartext
4. Valider les certificats SSL/TLS côté client
