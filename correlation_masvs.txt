VULNÉRABILITÉ #1 - DEBUG ENABLED
═══════════════════════════════════════════════════════════════
Titre: Application debuggable en production
Référence MASVS: MSTG-RESILIENCE-2
Catégorie: Reverse Engineering and Tampering
Description MASVS: "The app prevents debugging and/or detects, 
                    and responds to, being debugged."

Preuve de non-conformité:
- Fichier: AndroidManifest.xml
- Configuration: android:debuggable="true"
- Localisation: <application> tag

Impact:
- Permet aux attaquants d'attacher un debugger
- Dumping de la stack trace en temps réel
- Analyse du comportement runtime
- Accès aux classes de débogage

Recommandation:
- Définir android:debuggable="false" en production
- Utiliser BuildConfig.DEBUG pour différencier debug/release
- Implémenter une détection de debugger runtime

VULNÉRABILITÉ #2 - HARDCODED SECRETS (CRITIQUE!)
═══════════════════════════════════════════════════════════════
Titre: Secrets hardcodés dans le code source
Référence MASVS: MSTG-STORAGE-14
Catégorie: Data Storage and Privacy
Description MASVS: "No sensitive data is included in backups 
                    or stored in the app binary."

Preuve de non-conformité:
Secret #1:
- Fichier: owasp/sat/agoat/AIChatActivity.java
- Ligne: ~27
- Code: private final String openAIApiKey = "sk-abcdef1234567890..."

Secret #2 & #3:

- Code: aws_access_key_id = "AKIAX56QKKOLPQ7G7ABC"
- Code: aws_secret_access_key = "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"

VULNÉRABILITÉ #3 - ALLOWBACKUP ENABLED
═══════════════════════════════════════════════════════════════
Titre: Sauvegarde de données non protégée
Référence MASVS: MSTG-STORAGE-8
Catégorie: Data Storage and Privacy
Description MASVS: "No sensitive data is included in backups 
                    generated by the mobile operating system."

Preuve de non-conformité:
- Fichier: AndroidManifest.xml
- Configuration: android:allowBackup="true"

Impact:
- Extraction des données via ADB backup
- Vol de SharedPreferences, databases, fichiers
- Accès aux tokens, sessions, cache
- Pas de chiffrement des backups

Recommandation:
- Définir android:allowBackup="false"
- OU implémenter BackupAgent personnalisé avec chiffrement
- Exclure les fichiers sensibles via backup rules

VULNÉRABILITÉ #5 - HTTP CLEARTEXT TRAFFIC
═══════════════════════════════════════════════════════════════
Titre: Communication non chiffrée
Référence MASVS: MSTG-NETWORK-1
Catégorie: Network Communication
Description MASVS: "Data is encrypted on the network using TLS. 
                    The secure channel is used consistently 
                    throughout the app."

Preuve de non-conformité:
- URL: http://demo.testfire.net
- Fichier: owasp/sat/agoat/TrafficActivity.java
- Protocole: HTTP (cleartext)

Impact:
- Interception du trafic réseau (MITM)
- Vol de données sensibles en transit
- Manipulation des réponses serveur
- Session hijacking possible

Recommandation:
- Migrer toutes les URLs vers HTTPS
- Configurer network_security_config pour bloquer cleartext:
  <network-security-config>
    <base-config cleartextTrafficPermitted="false" />
  </network-security-config>
- Implémenter certificate pinning pour endpoints critiques

VULNÉRABILITÉ #6 - CRYPTOGRAPHIE FAIBLE (MD5)
═══════════════════════════════════════════════════════════════
Titre: Utilisation d'algorithme cryptographique faible
Référence MASVS: MSTG-CRYPTO-4
Catégorie: Cryptography
Description MASVS: "The app uses cryptographic primitives that 
                    are appropriate for the particular use-case."

Preuve de non-conformité:
- Fichier: owasp/sat/agoat/AccessControlIssue1Activity.java
- Algorithme: MD5
- Usage: Hash

Impact:
- MD5 est cassé (collisions connues)
- Ne convient pas pour hashing de passwords
- Ne convient pas pour intégrité cryptographique
- Vulnérable aux rainbow tables

Recommandation:
- Utiliser SHA-256 minimum pour hashing
- Utiliser bcrypt/scrypt/Argon2 pour passwords
- Utiliser HMAC pour intégrité des messages
- Ne JAMAIS utiliser MD5 ou SHA-1
