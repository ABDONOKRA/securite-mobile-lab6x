🔴 #1 - HARDCODED AWS CREDENTIALS (CATASTROPHIQUE)
═══════════════════════════════════════════════════════════════
Sévérité: CRITIQUE
CVSS Score: 9.8 (Critical)
Référence MASVS: MSTG-STORAGE-14
Référence CWE: CWE-798

Description:
Credentials AWS (Access Key ID + Secret Access Key) hardcodées 
en clair dans le code source Java.

Preuve:
- Fichier: owasp/sat/agoat/CloudServicesActivity.java
- aws_access_key_id = "AKIAX56QKKOLPQ7G7ABC"
- aws_secret_access_key = "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"

Impact:
- Accès TOTAL au compte AWS
- Création/suppression de ressources cloud
- Vol de données (S3, RDS, DynamoDB)
- Cryptomining aux frais du propriétaire
- Factures potentiellement ÉNORMES
- Compromission totale de l'infrastructure

Exploitation:
1. Décompiler l'APK avec apktool/jadx
2. Extraire les credentials
3. Configurer AWS CLI avec les credentials
4. Accès complet aux ressources AWS

Remédiation URGENTE:
1. RÉVOQUER IMMÉDIATEMENT ces credentials (AWS IAM Console)
2. Auditer CloudTrail pour détecter utilisation malveillante
3. Vérifier les factures AWS
4. Implémenter AWS Cognito/STS pour credentials temporaires
5. Utiliser AWS Secrets Manager
6. Ne JAMAIS stocker de credentials dans le code

🟠 #3 - DEBUG MODE ACTIVÉ EN PRODUCTION (ÉLEVÉ)
═══════════════════════════════════════════════════════════════
Sévérité: ÉLEVÉE
CVSS Score: 7.5 (High)
Référence MASVS: MSTG-RESILIENCE-2

Description:
L'application est compilée en mode debug (debuggable=true).

Preuve:
- AndroidManifest.xml: android:debuggable="true"

Impact:
- Attachment de debugger par des attaquants
- Dumping de la stack trace
- Analyse du comportement runtime
- Accès aux classes de débogage
- Facilite le reverse engineering

Remédiation:
- Définir android:debuggable="false" en production
- Utiliser BuildConfig.DEBUG pour différencier debug/release
- Implémenter une détection de debugger runtime

🟠 #4 - COMPOSANTS EXPORTÉS SANS PROTECTION (ÉLEVÉ)
═══════════════════════════════════════════════════════════════
Sévérité: ÉLEVÉE
CVSS Score: 7.0 (High)
Référence MASVS: MSTG-PLATFORM-1

Description:
Multiples composants Android exportés sans protection par permission.

Preuve:
- Service: DownloadInvoiceService (exported=true, 100%)
- Receivers: ShowDataReceiver (exported=true, 100%)
- Provider: ContentProviderActivity (exported=true)
- Activity: AccessControl1ViewActivity (exported=true)

Impact:
- Accès non autorisé par applications tierces
- Bypass des contrôles d'accès
- Injection de données malveillantes
- Fuite de données via Content Provider
- Déni de service

Remédiation:
- Définir exported="false" pour composants internes
- Ajouter android:permission pour composants devant être accessibles
- Implémenter validation stricte des intents reçus
- Utiliser signature-level permissions

Sévérité: MOYENNE

Référence MASVS: MSTG-NETWORK-1
